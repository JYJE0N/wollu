# 🚀 Korean Typing Master - 개발 명세서

## 📋 프로젝트 개요

**프로젝트명**: Korean Typing Master
**목적**: 한글/영문 타자연습 웹앱 (크로스 플랫폼 IME 지원)
**기술 스택**: Next.js 14 + TypeScript + Tailwind CSS + PostgreSQL + Zustand
**패키지 매니저**: Yarn
**핵심 라이브러리**: hangul-js, es-hangul, korean-js (한글 IME 처리)

## 🏗️ 아키텍처 설계 원칙

### 1. 관심사 분리 (Separation of Concerns)

```
📁 Core/     - 순수 비즈니스 로직 (IME, 타자측정, 티어시스템)
📁 UI/       - React 컴포넌트 및 스타일링
📁 Data/     - 상태관리 및 데이터베이스
📁 Services/ - 외부 API 및 유틸리티
```

### 2. 단일 책임 원칙 (Single Responsibility)

- **TypingEngine**: 타자연습 로직 조율
- **IMEHandler**: IME 이벤트 처리
- **TierSystem**: 티어 관리
- **ModuleRegistry**: 텍스트 모듈 관리
- **ShortcutManager**: 키보드 숏컷 관리

### 3. 중앙 집중식 상태 관리

- **Zustand Store**: 모든 상태를 중앙에서 관리
- **EventBus**: 컴포넌트 간 통신
- **Command Pattern**: 액션 처리

## 🔧 핵심 시스템 명세

### 1. 모듈형 텍스트 시스템

#### 텍스트 모듈 구조

```typescript
interface TextModule {
  id: string                    // 고유 식별자 (예: "ko-medium-philosophy")
  name: string                  // 모듈명
  description: string           # 모듈 설명
  language: 'ko' | 'en'        # 언어
  type: 'words' | 'sentences'  # 타입
  category: string              # 카테고리 (philosophy, literature, modern 등)
  difficulty: 'easy' | 'medium' | 'hard' # 난이도
  tags: string[]               # 태그 배열
  version: string              # 버전
  source: 'original' | 'public_domain' | 'inspired' # 출처
  metadata: ModuleMetadata     # 메타데이터
  content: ModuleContent       # 실제 콘텐츠
}
```

#### 연습 텍스트 분류 체계

```
단어 연습:
- 개수: 5개, 10개, 15개, 30개, 50개, 100개
- 언어: 한국어(ko) / 영어(en) 토글 제어
- 난이도: easy, medium, hard
- 카테고리: daily, modern, tech, academic, emotion, nature

문장 연습:
- 개수: 1개, 3개, 5개, 10개
- 길이: short(15자), medium(25~30자), long(30~50자)
- 타입: general, punctuation, numbers, mixed
- 주제: philosophy, literature, emotion, social, nature, meme, generational
```

```prisma
model User {
  id              String    @id @default(cuid())
  email           String    @unique
  name            String?
  currentTier     TierLevel @default(BRONZE)
  tierPoints      Int       @default(0)
  preferredLanguage String  @default("ko")

  practices       Practice[]
  tierAttempts    TierAttempt[]
  settings        PracticeSettings?
}

model Practice {
  id          String   @id @default(cuid())
  userId      String
  textId      String
  textContent String   # 실제 연습한 텍스트
  wpm         Int
  accuracy    Float
  errors      Json     # ErrorDetail[]
  keystrokes  Json     # Keystroke[]
  duration    Int      # milliseconds
  completedAt DateTime @default(now())
  language    String   # 'ko' | 'en'

  user User @relation(fields: [userId], references: [id])
}

model PracticeSettings {
  id             String @id @default(cuid())
  userId         String @unique

  # 단어 연습 설정
  wordCount      Int    @default(10)
  wordDifficulty String @default("easy")
  wordCategory   String @default("daily")

  # 문장 연습 설정
  sentenceCount  Int    @default(3)
  sentenceLength String @default("medium")
  sentenceType   String @default("general")

  # 공통 설정
  language       String @default("ko")
  theme          String @default("light")

  user User @relation(fields: [userId], references: [id])
}
```

## 🔄 데이터 흐름

### 타자연습 플로우

```
1. 사용자 Header Toolbar에서 설정 변경
   ↓
2. PracticeStore에서 TextGenerator 호출
   ↓
3. ModuleRegistry에서 조건에 맞는 모듈 검색
   ↓
4. SmartTextSelector가 개인화된 텍스트 추천
   ↓
5. TypingEngine이 IME 이벤트 처리
   ↓
6. 실시간 통계 계산 및 UI 업데이트
   ↓
7. 프랙티스 완료 시 Results 페이지로 이동
```

### 상태 관리 플로우

```
UI Event → Store Action → Core Logic → Database → UI Update
```

### 2. 상태 관리

- **Zustand**: 가벼움, TypeScript 친화적, 보일러플레이트 최소화
- **Persist 미들웨어**: 사용자 설정 영속화
- **Devtools**: 개발 시 상태 디버깅

### 3. 스타일링 전략

- **Tailwind CSS**: 반응형, 유틸리티 퍼스트
- **CSS Variables**: 테마 시스템 (동적 색상 변경)
- **Radix UI**: 접근성 보장된 기본 컴포넌트

### 프로젝트 주요 철학

1. 사용자 친화적
2. 사용자가 난이도를 설정할 수 있음
   1. 단어 문장의 타입, 길이를 선택
   2. 난이도가 어려울수록 티어 상승 가중치 적용
3. 사용자가 테마를 설정할 수 있음
4. 로그인을 하지 않아도 이용에 제한이 없음
5. 연습 예제(단어 또는 문장)의 품질: 사용자가 공감할 수 있는 문장과 다양한 레퍼런스.
   물고기를 잡아주는 것보다 물고기를 잡는 방법을 알려주는 것이 낫다.
   늦었다고 생각할 때가 정말 늦었다.
   사랑은 타이밍.
